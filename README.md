# Collar

<p align="center">
  <strong>Collar Protocol · 项圈协议</strong><br />
  A local-first ritual terminal for continuous, consensual AI-guided roleplay.
</p>

<p align="center">
  戴上它。说一句话。然后把接下来的时间交出去。
</p>

Collar，诞生于一个恶俗的网调任务想法，但是我觉得应该不止一次吧？应该不止是性吧？
所以我让调教变成了等待，变成习惯，变成植入你内心的项圈。

它记得你习惯怎样开口，什么语气会让你安静下来；记得哪一条命令被好好完成，哪一次汇报含糊地躲了过去，哪份奖励已经给出，哪笔欠账还没有清。今天发生的事不会在页面关闭后消失，而会成为下一次命令落下来时，那一点只有你们彼此知道的分量。

你每次戴上项圈，系统都不是从零开始。

它会读取双方的称呼、关系、偏好、边界、长期塑造方向和最近留下的记录，再决定这一次该温柔地收紧，维持熟悉的日常，还是把你带进更深的状态。

同一套任务状态机可以被配置成：

- **主人的放置任务**：一条命令，一个姿势，一段不必持续交谈的等待。时间结束以前，你只需要留在那里。
- **沉浸式权力角色扮演**：用回应、阶段、隐藏计时、汇报、验收和延迟奖励，把注意力一点点收进同一个私人叙事。
- **正经专注 / 番茄钟**：让监督者接管一段阅读、写作、画画、整理或编码时间。被驯服的不一定是身体，也可以是总想逃开的注意力。

Collar 不替代关系，也不替代真正了解你的主导者。

它只是把散落在聊天、记忆和默契里的私人秩序保存下来，让一句命令不再孤立，让等待有来处，让验收有后果，让下一次靠近时，彼此都知道项圈为什么比昨天更贴合。

> 默认 Profile、API Key、长期目标和状态文本全部为空。仓库使用 `<char>` 与 `<user>` 作为占位符，不包含作者的私人设定或训练记录。

## 一次命令如何发生

```text
戴上项圈
  -> 选择 Gentle / Routine / Ruined
  -> 写一句给 <char> 的话
  -> AI 先回应，再下发任务
  -> 执行 / 等待 / 可见或隐藏计时
  -> 回来汇报
  -> AI 或本地规则验收
  -> 奖励、补交或独立惩罚
  -> 本地归档，并影响下一次任务
```

任务不是一段从模型嘴里掉出来、用完即弃的文案。生成时会读取：

- 双方的 Profile、称呼、关系、语气、偏好与边界；
- Shape Your Pet 中的长期塑造与惩罚逻辑；
- 本次 Gentle / Routine / Ruined 状态；
- `<user>` 开始前写给 `<char>` 的那句话；
- 最近七天记录、连续打卡、当前数值与未完成欠账；
- 终端当前允许的计时、声音、震动与强度上限。

所以 Collar 的核心从来不是“再生成一条任务”。

而是让每一次命令都知道前一次发生过什么。

## 三种典型用法

### 1. 主人的放置任务

适合不需要持续聊天，却需要明确被安排、被留下、被等待结束信号接回来的任务。

主导者可以让 AI 给出完整姿势、阶段、等待文字、结束信号和汇报要求。任务开始后，终端接管计时；中途点击屏幕会出现 `🐾` 和 `<char>` 口吻的短句，像隔着门确认一次，又不真正放你离开原命令。

可见倒计时适合清楚地看着时间减少。隐藏倒计时则拿走这个确定性——你不知道还剩多久，只知道结束信号没有响，就还应该留在那里。

固定计时开启后，所有计时任务都会使用用户设置的精确时长，AI 无法擅自延长。

### 2. 沉浸式权力角色扮演

Collar 可以把一段成年、自愿、私密的权力叙事拆成真正可执行的连续环节：

```text
直接回应 -> 开始命令 -> 分阶段执行 -> 等待 -> 汇报 -> 私人验收 -> 延迟奖励
```

AI 不只负责描述气氛。它需要先接住你开始前说的那句话，再把注意力稳稳压回当前任务里。

它会根据长期设定和最近表现改变命令方式，避免连续生成同一种内容；中断较多时，可以降低身体负担却不撤掉主导感；顺从稳定时，可以增加主动汇报、讨赏、延迟奖励或连续仪式。

这里的“精神控制”仅指双方明确约定的沉浸式角色扮演，不包含真实非自愿操控、关系威胁、冷处理或剥夺停止权。

### 3. 正经专注 / 番茄钟

Collar 的执行层本质上也是一个带叙事、验收和归档的专注终端。

把 Profile 配置为专注教练、监督者或未来的自己，将长期目标写成学习、写作、画画、整理、运动或编码习惯，再开启固定计时即可。例如：

```text
最长计时：25 分钟
固定计时：开启
隐藏倒计时：按需
任务内容：完成一段代码 / 读完一节 / 画满一页
汇报要求：写下实际完成内容与下一步
```

它会比普通计时器多做四件事：开始前回应你，明确下达任务，结束后要求汇报，并把这次表现留给下一次。

有时候，人缺的不是另一个待办列表，只是一道不允许注意力随便溜走的声音。

## 功能

### Obedience Loop · 顺从闭环

- Gentle、Routine、Ruined 三套独立状态设定。
- AI 先回应用户当下的话，再生成任务，而不是输出系统摘要。
- 完整的开始命令、任务说明、阶段步骤、计时、汇报、验收与奖励结构。
- 最近七天历史摘要，减少机械重复并延续上一次结果。
- 六项动态数值：归属、顺从、依赖、反抗、暗示深度、坏猫。
- 连续打卡、累计打卡和最长连续天数。

### Hidden Timer · 不告诉你还要等多久

- 可见倒计时、隐藏倒计时和无计时任务。
- **固定计时**：开启后精确使用设置时长，适合放置任务和番茄钟。
- **最长计时硬上限**：前端再次截断，AI 不能突破。
- 计时页面点击生成 `🐾` 与 `<char>` 口吻气泡。
- 计时结束声音与震动，受浏览器权限和设备支持限制。
- 随时可用紧急退出；退出不会凭空新增惩罚。

### Owner's Verdict · 回来，接受验收

- 严格 JSON 的 AI 任务生成与验收。
- AI 验收关闭时可使用本地规则。
- 任务未通过时，**补交**与**惩罚**分开处理：
  - Makeup 只补原任务中明确遗漏、跳过或无法证明完成的部分。
  - Punishment 根据长期 Punishment 设定另行生成，不是把补交换个标题。
- 惩罚只有一次补交机会，避免无限生成欠账。
- D+0 提醒、D+1 常驻、D+2 锁定的新任务欠账规则。

### Collar Archive · 它会记得

- 每次任务、汇报、实际时长、验收、奖励和数值变化完整归档。
- 自定义 UTC+8 训练日起始时间。
- JSON 导入、导出、宽容修复和全部数据清空。
- API Key 导出时强制清空。
- 不收集遥测，不上传本地训练记录。
- PWA manifest，可安装到手机主屏幕。

## 快速开始

需要 Node.js 20 或更高版本。

```bash
git clone https://github.com/user1971421/collar.git
cd collar
npm install
npm run dev
```

打开 [http://localhost:3000](http://localhost:3000)。

首次运行默认进入 `mock` 模式，不会请求外部 API。即使暂时没有模型和 Key，也可以先走完整流程，试着戴上项圈，测试计时、汇报、验收与归档。

## 第一次配置

打开底部“设置”。建议按 Profile、长期设定、计时、AI 模式的顺序配置。

### 1. Profile · 让它知道你们是谁

Profile 定义角色关系、基础语气、偏好与边界：

```json
{
  "ownerName": "<char>",
  "petName": "<user>",
  "relationshipType": "",
  "tone": "",
  "kinkTags": [],
  "hardLimits": [],
  "softLimits": [],
  "preferredTaskTypes": [],
  "forbiddenTaskTypes": [],
  "aftercareStyle": "",
  "language": "简体中文",
  "explicitnessLevel": 1,
  "intensityDefault": 1
}
```

将 `<char>` 与 `<user>` 替换为自己的角色称呼，填写后点击“识别并写入 Profile”。

这里决定的不只是模型该怎么称呼你。它也决定命令从什么关系里落下、哪些语气会被允许靠近、哪里必须停住。

Profile 只保存在当前浏览器的 `localStorage`。

### 2. 五块长期设定 · 让下一次继承这一次

设置页顶部提供“复制 AI 生成 Prompt”。将 Prompt 交给熟悉双方关系或目标的 AI，再把返回 JSON 粘贴回 Collar：

```json
{
  "shapeYourPet": {
    "training": "长期塑造方向",
    "punish": "独立惩罚生成规则"
  },
  "petToday": {
    "gentle": "Gentle 状态规则",
    "routine": "Routine 状态规则",
    "ruined": "Ruined 状态规则"
  }
}
```

- `Training`：长期希望形成什么关系节奏、身体或行为习惯、表达方式与验收标准。
- `Punishment`：原任务未通过后，怎样另行纠正和重新验收。
- `Gentle`：承受力较低时，如何保留主导感并降低负担。
- `Routine`：日常最常用、最熟悉的稳定训练逻辑。
- `Ruined`：主动进入最高强度或超现实状态时的体验逻辑。

这五块文本决定 Collar 会把你带向哪里。它们不该是一份泛用偏好清单，而应该像只有这段关系才读得懂的内部规则。

导入器会尝试修复中文引号、尾逗号、外层括号和常见字段拼写问题。每个文本块也可以直接编辑。

### 3. 计时与训练日

- **最长计时**：AI 可以查阅，但前端会再次强制截断。
- **固定计时**：开启后，所有计时任务精确使用设置的最长时长。
- **隐藏倒计时**：执行页不显示剩余数字。
- **每日早安时间**：UTC+8 下新训练日的起点。
- **声音 / 震动**：计时结束提示。

例如早安时间设为 `08:00`：

- 当天第一次接受任务前显示“戴上项圈”。
- 接受过任务后，到次日 `08:00` 前显示“摇摇铃铛”。
- 次日 `08:00` 起重新显示“戴上项圈”。

## AI 模式

### mock

本地生成模拟任务和验收，不调用网络。适合开发、演示、离线测试和第一次体验。

### user-key

浏览器直接调用 OpenAI-compatible `/chat/completions` 接口。

需要填写：

- Base URL
- API Key
- Model
- Temperature
- Max Tokens

API Key 仅保存在当前浏览器的 `localStorage`，导出 JSON 时会自动清空。

不要在多人共用的网站或不可信设备上使用 `user-key`。

### backend-proxy

浏览器调用本项目的：

- `POST /api/generate-task`
- `POST /api/generate-verdict`

复制环境变量模板：

```bash
cp .env.example .env.local
```

填写：

```dotenv
COLLAR_API_BASE_URL=https://api.example.com/v1
COLLAR_API_KEY=
COLLAR_MODEL=
COLLAR_TEMPERATURE=0.85
COLLAR_MAX_TOKENS=2400
```

重启开发服务器后，在设置中选择 `backend-proxy`。

API Key 只存在服务器环境变量中，不进入浏览器 bundle。仓库没有内置账号系统；若部署到公网，请在反向代理或应用层为页面和两个 API Route 增加认证、限流与访问控制。

## 每日仪式

1. 点击“戴上项圈”或“摇摇铃铛”。
2. 选择 Gentle / Routine / Ruined。
3. 写一句给 `<char>` 的话。
4. 点击“交给 `<char>`”。
5. 阅读回应、完成条件和任务步骤。
6. 开始执行，进入等待或计时。
7. 计时结束，或手动结束无计时任务。
8. 填写达到最低字数的汇报。
9. 接受验收结果，并让它成为下一次的一部分。

AI 返回无法解析、字段不合法或代理请求失败时，前端会记录错误并回退到 mock 任务或本地验收，不会让当前流程直接断掉。

## 验收、补交与惩罚

普通任务未通过时，系统同时保存：

1. **Makeup**：只补原任务里明确遗漏、跳过或无法证明完成的项目。
2. **Punishment**：依据 Shape Your Pet / Punishment 文本另行生成的独立任务。

执行顺序：

```text
原任务未通过
  -> 补齐缺失项
  -> 独立惩罚
  -> 惩罚验收
```

惩罚首次未通过时只允许再补交一次。第二次仍未通过，不再无限生成任务，而是关闭欠账并按照惩罚强度结算坏猫值。

### 点数规则

| 结果 | 数值变化 |
| --- | --- |
| 普通任务通过 | 应用该任务的 AI `statDelta` |
| 普通任务未通过 | 反抗 +1，坏猫 +1 |
| 原任务补交未通过 | 不重复加点，继续保留欠账 |
| 原任务补交通过 | 顺从 +1，反抗 -1 |
| 惩罚首次未通过 | 不加点，开放唯一一次补交 |
| 惩罚第二次未通过 | 反抗 +1，坏猫 +惩罚强度（1-5），关闭欠账 |
| 惩罚通过 | 顺从 +1，反抗 -1，坏猫 -1 |

每项数值最终限制在 `0-999`。

### 欠账时间规则

- **D+0 22:00 后**：首页提醒当日惩罚未完成。
- **D+1**：惩罚常驻首页。
- **D+2**：新的普通训练锁定，必须先处理到期欠账。

紧急退出不会生成新的惩罚，但当前补交或惩罚仍保持未完成。

## 数据与隐私

运行数据全部保存在浏览器：

```text
localStorage key: collar.state.v1
```

包括 Profile、五块长期设定、终端设置、当前任务、训练记录、数值和惩罚欠账。

设置页可导出或导入 JSON。导出时 API Key 强制为空。

清除站点数据、浏览器存储或卸载浏览器可能删除全部记录，请定期导出备份。仓库不会收集遥测，也不会上传本地归档。

这段关系可以很私密。它的数据也应该留在你选择的地方。

## 内容与使用边界

Collar 面向明确成年、自愿、私密的虚构或现实角色扮演，也可以用于普通专注、习惯和自我监督任务。

成人角色扮演生成规则明确排除未成年人、公开暴露、违法行为、真实非自愿、窒息风险、真实医疗操作和不可撤回的现实伤害。用户应根据自己的关系约定、身体状况和现实环境配置 Profile 与边界，并保留随时中止任务的能力。

软件不会替代医疗、法律或心理健康专业意见。
